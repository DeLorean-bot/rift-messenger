# 45 — Темизация (Theming)

Темизация — это способность менять внешний вид клиента (цвета, шрифты, плотность, эффекты)
без изменения кода. У Discord есть два уровня: **официальные темы/настройки внешнего вида**
и **пользовательские темы** (через клиент-моды вроде BetterDiscord — неофициально).

> Референс к этому разделу: пользователь предоставил тему **«Discord 11»** (zuzumi-f) —
> BetterDiscord `.theme.css` и Stylus `.user.css`. Ниже она разобрана как показательный
> пример того, какой механизм темизации нужно спроектировать в клоне.

---

## 45.1. Модель темизации через CSS-переменные

### Что делает
Вся палитра, размеры и эффекты интерфейса управляются **CSS custom properties** (переменными)
в `:root`. Компоненты не хардкодят цвета — они ссылаются на переменные. Смена темы = замена
значений переменных.

### Как работает / ожидаемое поведение
```css
:root {
  /* базовая палитра (пример) */
  --background-primary:   #313338;  /* фон чата */
  --background-secondary: #2b2d31;  /* сайдбар каналов */
  --background-tertiary:  #1e1f22;  /* лента серверов, самый тёмный */
  --text-normal:          #dbdee1;
  --text-muted:           #949ba4;
  --brand-500:            #5865f2;  /* blurple / accent */
  --status-online:        #23a55a;
  --status-idle:          #f0b232;
  --status-dnd:           #f23f42;
  --radius-md:            8px;
}
```
Компонент: `background: var(--background-primary)`. Тёмная/светлая/Midnight-темы — это разные
наборы значений тех же переменных. Пользовательская тема просто переопределяет `:root`.

### Что происходит на клиенте
- Активная тема применяет набор переменных на корень документа (атрибут/класс `theme-dark`,
  `theme-light`, `data-theme`).
- Переключение темы — мгновенная перерисовка (браузер пересчитывает `var()`), без перезагрузки.
- Accent-color пользователя (Nitro/настройки) переопределяет `--brand-*`.

### Что происходит на сервере
- Ничего в рендере: тема — чисто клиентская. Сервер лишь **хранит выбор** (`theme`,
  `accent_color`) в настройках пользователя (`USER_SETTINGS_UPDATE`) для синхронизации
  между устройствами (см. 01.12).

### Технологии
CSS custom properties + `prefers-color-scheme` для авто-режима.

### Альтернативы и почему так
- **Пре-компилированные темы (SCSS в отдельные CSS-файлы)** — быстрее в рантайме, но нельзя
  менять на лету и нельзя дать пользователю кастомный accent. CSS-переменные дают
  живое переключение и пользовательскую настройку одним механизмом.
- **CSS-in-JS с темой в JS-объекте** — гибко, но перерисовка идёт через React; CSS-переменные
  меняют вид без ре-рендера дерева, что дешевле на большом UI.

## 45.2. Встроенные темы и настройки внешнего вида

### Что делает
Даёт пользователю готовые режимы без правки CSS.

### Ожидаемое поведение (вкладка Appearance)
- **Theme:** Dark / Light / Midnight (OLED-чёрная) / Sync with system.
- **Accent color / Nitro theme:** переопределение бренд-переменных.
- **Message display:** Cozy (аватар + отступы) / Compact (одна строка, IRC-стиль).
- **Font scaling / Zoom / Space between message groups** — переменные размеров.
- **Saturation** — глобальная насыщенность (для дальтонизма/комфорта).
- **Reduced motion** — отключает/смягчает анимации (см. главу 14 и 46).

### Технологии
Значения из `UserSettingsStore` → маппинг в переменные/классы на корне.

## 45.3. Пользовательские темы (клиент-моды) — на примере «Discord 11»

> Официально Discord не поддерживает произвольный пользовательский CSS; это делают
> сторонние моды (BetterDiscord, Vencord, Stylus). Клон **может** предусмотреть безопасный
> API тем — раздел показывает, как это устроено на практике.

### Разбор предоставленных файлов

**`Discord11.theme.css` (формат BetterDiscord):**
- Метаданные в комментарии-заголовке: `@name`, `@description`, `@author`, `@version`,
  `@source`. BetterDiscord парсит их для списка тем.
- Тело — набор `@import url(...)`, подгружающих реальные стили с CDN автора
  (`base.css`, `messagebubble11.css`, `windows11font.css`, `glasseffect.css`) + замена
  emoji-набора на Microsoft.
- Блок `:root` с **пользовательскими настройками** темы:
  ```css
  --wallpaper-is: none;     /* фоновое изображение */
  --blur-size: 24px;        /* сила размытия (acrylic) */
  --opacity-thing: 0.6;     /* прозрачность панелей */
  --server-size: 48px;      /* размер иконок серверов */
  --H: 199; --S: 100%; --L: 69%;  /* accent-цвет в HSL */
  ```
  Accent задаётся **в HSL-компонентах**, а не готовым hex — тема потом собирает цвета как
  `hsl(var(--H) var(--S) var(--L))`. Это ключевой приём: одна тройка H/S/L перекрашивает
  все акцентные элементы.

**`Discord11.user.css` (формат Stylus/UserCSS):**
- Заголовок `==UserStyle==` с `@preprocessor stylus` и **декларативными настройками**:
  ```
  @var checkbox messageBuble  "Enable Message Bubble Style" 1
  @var checkbox win11Emojis   "Enable Windows 11 emojis" 1
  @var checkbox glassEffect   "..." 0
  @var text     wallpaperImage "..." '...'
  @var number   hueAmount      "Hue amount" [199, 0, 239, 1]
  @var range    saturationAmount "..." [100, 0, 100, 1, '%']
  ```
  Stylus генерирует **UI настроек темы** из этих `@var` (чекбоксы, ползунки, поля) — user
  крутит их, значения подставляются в CSS.
- `@-moz-document regexp("https?://(canary\.)?discord.com/.*")` — область применения: стиль
  инжектится только на домены Discord.
- Условные `if messageBubble==default { @import ... }` — включение частей по галочкам.

### Чему учит этот пример (выводы для клона)
1. **Тема = переопределение `:root`-переменных** + опционально подгружаемые модули стилей.
2. **Accent в HSL-компонентах** — правильный дизайн-токен: даёт пользователю живой ползунок
   оттенка/насыщенности/яркости без пересбора палитры.
3. **Настройки темы декларативны** (checkbox/number/range/text) и генерируют UI — клон может
   принять такой же манифест для «магазина тем».
4. **Эффекты платформы** (acrylic-blur, wallpaper, прозрачность) выражаются переменными
   `--blur-size`/`--opacity-thing` — их стоит завести как first-class токены, если целишься
   в «Windows 11 / стеклянный» вид.
5. **Замена шрифта и эмодзи** — отдельные независимо включаемые слои.

### Ожидаемое поведение системы тем в клоне
- Тема — это манифест (метаданные + список настроек + CSS/переменные).
- Настройки темы сохраняются локально; сама тема применяется инъекцией `<style>` в документ.
- Область применения ограничена доменом приложения.
- Небезопасный произвольный CSS изолируется (не даёт доступа к JS/Node — см. 01.6);
  темы не должны уметь выполнять код.

### Альтернативы и риски
- **Полный произвольный CSS от пользователя** — максимальная гибкость, но: (а) ломается при
  каждом изменении вёрстки (селекторы «уплывают»), (б) риск фишинга (тема перерисовывает
  диалоги ввода пароля). Поэтому официальный путь — **ограниченный набор токенов/тем**, а не
  сырой CSS; клиент-моды берут риск на себя.
- **Именованные семантические токены** (`--background-primary`) устойчивее к рефакторингу
  вёрстки, чем стилизация по конкретным CSS-классам компонентов (как делает «Discord 11»,
  цепляясь за внутренние классы Discord, из-за чего темы регулярно ломаются на обновлениях).

## 45.4. Рекомендуемый набор дизайн-токенов (для rift-messenger)

Минимальный устойчивый контракт переменных, на который должны ссылаться все компоненты:

```css
:root {
  /* Поверхности (от светлого к тёмному слою) */
  --surface-1: …;  /* лента серверов */
  --surface-2: …;  /* сайдбар каналов */
  --surface-3: …;  /* фон чата */
  --surface-input: …;
  --surface-floating: …;  /* поповеры, меню, модалки */

  /* Текст */
  --text-normal: …; --text-muted: …; --text-link: …;
  --text-positive: …; --text-danger: …;

  /* Бренд/accent — в HSL-компонентах для живой настройки */
  --accent-h: 235; --accent-s: 86%; --accent-l: 65%;
  --accent: hsl(var(--accent-h) var(--accent-s) var(--accent-l));

  /* Статусы */
  --status-online: …; --status-idle: …; --status-dnd: …; --status-offline: …;

  /* Форма */
  --radius-sm: 4px; --radius-md: 8px; --radius-lg: 16px; --radius-round: 50%;

  /* Плотность/размеры */
  --server-icon-size: 48px;
  --sidebar-width: 240px;
  --members-width: 240px;

  /* Эффекты (опционально, для glass/acrylic) */
  --blur-size: 24px;
  --panel-opacity: 1;
  --wallpaper: none;

  /* Анимация */
  --motion-fast: 100ms; --motion-med: 200ms; --motion-slow: 300ms;
  --motion-ease: cubic-bezier(0.4, 0, 0.2, 1);
}
```

Полная палитра значений (light/dark/midnight) — в главе **48 (Дизайн-система)**.

Связанные разделы: **14 (Анимации)**, **46 (Accessibility — reduced motion, контраст)**,
**48 (Дизайн-система)**, **01.6 (изоляция клиента и безопасность инъекций)**.
