# RIFT Messenger

Экспериментальный P2P-мессенджер без центрального сервера. Текстовые сообщения,
камера, микрофон и демонстрация экрана передаются напрямую через WebRTC.

## Что уже работает

- прямой текстовый чат между двумя устройствами;
- микрофон, камера и демонстрация экрана;
- создание и переключение текстовых каналов;
- автоматическая передача нового канала собеседнику с первым сообщением;
- P2P-передача изображений, видео, аудио и других файлов до 8 МБ;
- подключение устройств через сжатые QR-коды без пересылки SDP-текста;
- сканирование QR с камеры или из сохранённой картинки;
- короткие одноразовые ссылки `rift://join/...` с автоматическим обменом offer/answer;
- зашифрованный signaling через несколько бесплатных публичных Nostr-реле;
- локальное хранение вложений в IndexedDB без облачного хранилища;
- локальный профиль пользователя;
- сохранение каналов, профиля и истории в браузере;
- счётчики непрочитанных сообщений.

## Запуск

```powershell
npm install
npm run dev
```

Откройте адрес из терминала в браузере. Для проверки на одном компьютере откройте
приложение в двух разных окнах браузера.

## Windows-приложение

Готовый установщик находится здесь:

```text
releases/RIFT_0.5.0_x64-setup.exe
```

Для новой сборки:

```powershell
npm run desktop:build
```

Чтобы связать два компьютера через интернет, установите RIFT на оба устройства.
Один пользователь создаёт короткую ссылку `rift://join/...` и отправляет её
другому. Второй открывает ссылку; RIFT автоматически забирает зашифрованное
приглашение и возвращает ответ через публичные Nostr-реле. `localhost` для этого
не используется, а после знакомства сообщения и медиапотоки идут напрямую.

## Как соединиться

1. Первый пользователь выбирает «Подключить друга» → «Создать приглашение».
2. Он отправляет полученный код второму пользователю.
3. Второй выбирает «Ввести приглашение», вставляет код и возвращает код ответа.
4. Первый вставляет код ответа и завершает соединение.

Ручной обмен кодами заменяет signalling-сервер. Используется публичный STUN только
для определения сетевого адреса. STUN не передаёт сообщения или видеопоток.

## Ограничения прототипа

- одно прямое соединение одновременно;
- история хранится локально и пока не синхронизируется между устройствами;
- соединение через строгий NAT может потребовать TURN-ретранслятор;
- камера и демонстрация экрана требуют безопасного контекста (HTTPS или localhost).
