# RIFT Messenger

Экспериментальный P2P-мессенджер в духе Discord — без центрального сервера.
Текст, голос, камера и демонстрация экрана идут напрямую между устройствами
через WebRTC.

## Стек

- **Electron** — десктоп-оболочка (Windows), нативные окна, `rift://` deep links,
  экранный захват (`desktopCapturer`) и подписанные автообновления.
- **React 18 + TypeScript + Vite** — интерфейс.
- **WebRTC** — прямые DTLS-SRTP аудио/видео/screen-share + RTCDataChannel для
  сообщений и файлов.
- **Nostr public relays** — временный зашифрованный rendezvous для коротких
  приглашений (offer/answer шифруются AES-GCM).
- **RNNoise (WASM) + WebRTC AEC/AGC/NS** — обработка микрофона.
- **electron-builder + electron-updater** — installer и GitHub-релизы.

## Что уже работает

**Голос (уровня Discord):**
- выбор устройств ввода/вывода, громкость микрофона и собеседника, deafen;
- режим ввода: активация по голосу (с чувствительностью) или push-to-talk;
- RNNoise / эхоподавление / автоусиление, проверка микрофона с метром;
- индикатор говорящего (active speaker), авто-reconnect при потере сети.

**Демонстрация экрана (in-app пикер как в Discord):**
- выбор окна или монитора с живыми превью (через `desktopCapturer`);
- разрешение до 1440p, 15/30/60 fps, выбор кодека (H264/VP9/AV1), звук приложения;
- смена разрешения/fps на лету без переподключения.

**Текст и связь:**
- прямой чат, текстовые каналы, счётчики непрочитанного;
- короткие ссылки `rift://join/...` с авто-обменом offer/answer через Nostr;
- P2P-передача файлов до 8 МБ, локальное хранение в IndexedDB;
- подключение рядом через сжатые QR-коды.

## Запуск (веб/дев)

```powershell
npm install
npm run dev
```

Откройте адрес из терминала. Для проверки на одном ПК откройте два окна браузера.

## Десктоп (Electron)

Запуск приложения в Electron:

```powershell
npm run desktop:dev
```

Собрать подписываемый installer + опубликовать релиз (electron-builder):

```powershell
npm run desktop:build
```

Чтобы связать два компьютера через интернет, поставьте RIFT на оба. Один создаёт
короткую ссылку `rift://join/...`, второй её открывает — приглашение подхватывается
автоматически через публичные Nostr-реле, после чего медиа идёт напрямую P2P.

## Ограничения прототипа

- одно прямое P2P-соединение одновременно (multi-user требует SFU-сервера);
- история хранится локально и не синхронизируется между устройствами;
- соединение через строгий NAT может потребовать TURN-ретранслятор.
